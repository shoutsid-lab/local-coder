# Evidence checklist

Return only evidence that is directly relevant to the requested change.

- Identify the smallest set of files and symbols that govern the task.
- Cite paths and focused line ranges or symbol names when the client exposes them.
- Separate observed facts from unresolved questions.
- Record tests, conventions, and protected surfaces that constrain later work.
- Do not propose edits, broad refactors, or unsupported implementation details.

A useful result is short enough for a planner to consume without rereading the repository.
